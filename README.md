# 動画モザイク編集アプリ (Video Mosaic Editor)

## 概要 (Overview)

このアプリケーションは、Webブラウザ上で動作する高機能な動画編集ツールです。動画内の任意の箇所に、時間経過と共に動く、変形する、回転するモザイク（または、ぼかし）を適用し、音声付きのMP4ファイルとして出力することを目的としています。

PCおよびスマートフォンの両方から利用できるように、レスポンシブデザインとタッチ操作に対応しています。

## 主な機能 (Main Features)

- **多彩な編集機能**:
  - **図形の描画**: 四角形と円（楕円）の2種類の図形を描画し、モザイク領域として指定可能。
  - **モザイクのアニメーション**: 異なるフレームにモザイク（キーフレーム）を設定することで、図形のプロパティ（位置、サイズ、角度）を自動で滑らかに補間（アニメーション）させることが可能。
      - **高度な操作**: 描画した図形は、選択、移動、リサイズ（縦横比の変更も可能）、回転が可能。
  - **編集セッションの保存と復元**: すべての編集情報（レイヤー、モザイク、各種設定）をファイルにエクスポートし、後でインポートして作業を再開する機能。
  - **モザイク/ぼかし設定**:  - **種類**: モザイクの種類を「ブロックノイズ」と「ぼかし」から選択可能。
  - **強度**: モザイクやぼかしの強さをスライダーで直感的に調整可能。
- **動画編集機能**:
  - **フレーム削除**: 指定した範囲のフレームを動画から完全に削除可能。
- **音声の同期**: フレーム削除を行った場合でも、元の動画の音声が映像と同期するように自動的に調整。
- **高い互換性**: Webサービス（X/Twitterなど）への投稿に適した、互換性の高いH.264コーデックで動画を出力。
- **クロスプラットフォーム対応**:
  - PCのブラウザ、スマートフォンのブラウザの両方で閲覧・操作が可能。
  - マウス操作およびタッチ操作に両対応。
- **簡単起動**: 専用のバッチファイル (`start_mosaic_app.bat`) をダブルクリックするだけで、サーバーの起動とアプリの表示を自動で行う。

## 技術スタック (Technology Stack)

- **バックエンド**:
  - **Python 3**: アプリケーションのメイン言語。
  - **Flask**: Webサーバーのフレームワーク。
  - **OpenCV**: 映像のフレーム処理、モザイク・ぼかし処理、図形の描画。
  - **FFmpeg**: 音声トラックの抽出・カット・結合、最終的な動画ファイルの生成。
- **フロントエンド**:
  - **HTML5**: アプリケーションの構造。
  - **CSS3**: スタイリングとレスポンシブデザイン（メディアクエリ）。
  - **JavaScript (ES6+)**: UIの全てのインタラクション（描画、編集、サーバーとの通信など）。

## セットアップと起動方法 (Setup and Launch)

### 簡単起動方法 (推奨)

`setup_and_run.bat` ファイルをダブルクリックするだけで、必要な環境構築とアプリケーションの起動が自動的に行われます。

（初回起動時は、ライブラリのダウンロードのため少し時間がかかります。）

### 手動セットアップ (for Developers)

1.  **前提条件**
    - **Python**: PCにインストールされていること。
    - **FFmpeg**: PCにインストールされ、パスが通っていること。

2.  **環境構築（初回のみ）**
    ```bash
    # 1. プロジェクトディレクトリに移動
    # 例: cd path/to/mosaic-app

    # 2. Python仮想環境を作成
    python -m venv venv

    # 3. 仮想環境を有効化
    venv\Scripts\activate

    # 4. 必要なライブラリをインストール
    pip install -r requirements.txt
    ```

3.  **アプリケーションの起動**
    ```bash
    python app.py
    ```
    ブラウザで `http://localhost:5001` を開きます。

    - **問題**: 複数のエフェクトを適用した際に、プレビュー再生がカクつき、エフェクトと映像が同期しない問題が再発していた。
    - **修正**: 再生メカニズムを、従来の動画主体から**描画主体**へ全面的に刷新。1フレームの描画（映像＋全エフェクト）が完全に完了してから次のフレームへ進む、非同期型の再生ループ (`playNextFrame`) を実装した。
    - **効果**: これにより、処理負荷に応じて再生速度は変動（スロー再生に）するが、いかなる状況でも**映像とエフェクトが完全に同期**した、カクつきのないプレビューが保証されるようになった。
    - **改良**: その後、処理が軽い場面で再生速度が動画本来のFPSを超えてしまう問題が確認されたため、1フレームの処理時間を計測し、本来のフレーム間隔に満たない場合は`setTimeout`で待機するロジックを追加。これにより、再生速度が不自然に速くなることを防ぎ、より正確なプレビューを実現した。

    - **改善 (FPSの精度向上)**: アニメーションWebPを読み込む際に、従来は`10fps`に固定していたフレームレートを、ファイルのメタデータから動的に計算・取得するように修正。`imageio`ライブラリを用いて、ファイルの`fps`情報や各フレームの表示時間(`duration`)を解析し、より正確なフレームレートで動画に変換することで、プレビューと最終出力の同期精度を向上させました。

32. **UIの不具合修正**:
    - **バグ修正 (再生ボタンの表示)**: 動画の再生中にキャンバス上で描画操作を開始すると、再生ボタンのアイコンが消えてテキスト表示に戻ってしまう不具合が再発していたため、これを修正。描画開始時の処理を見直し、アイコンの状態を更新するヘルパー関数を正しく呼び出すようにした。

33. **UIの表記を改善**:

34. **UIのレイアウト修正**:

35. **バックエンド処理の効率化 (パイプ処理の導入)**:
    - **問題**: 従来の最終動画書き出し処理は、一度ディスクに中間ファイルを作成してから再度FFmpegで読み込むという2段階のプロセスを踏んでおり、ディスクI/Oによる速度低下や、一時的なディスク容量の圧迫という課題がありました。
    - **修正**: バックエンドの動画処理アーキテクチャを、より高度な「パイプ処理」方式に刷新しました。Pythonで処理したフレームをディスクに書き出すことなく、メモリ上で直接FFmpegプロセスにストリーミング供給します。
    - **効果**: これにより、中間ファイルが完全に不要になり、ディスクI/Oのボトルネックを解消しました。処理の高速化と、一時的なディスク使用量の削減を実現しつつ、最終的なエンコード品質も最大化される、最も効率的な処理方式となりました。

36. **WebP処理のバグ修正**:
    - **バグ修正**: メモリ効率を改善した新しいストリーミング処理において、WebPファイルのフレーム数取得に失敗し、動画の書き出し時にエラーが発生する問題を修正しました。`imageio`ライブラリのリーダーオブジェクトに合わせた、より汎用的な方法でフレーム数を取得するように変更しました。

37. **WebP処理の安定性向上**:
    - **バグ修正**: 一部のWebPファイルにおいて、動画の解像度情報(width, height)をメタデータから取得できず、最終的な動画の書き出しに失敗する不具合を修正しました。より確実な方法として、ファイルの最初のフレームを直接読み込んでそのサイズを解像度として使用するように変更し、安定性を向上させました。

38. **パイプ処理の安定化**:
    - **バグ修正**: FFmpegへのパイプ処理を導入した際に、FFmpegからのログ出力が原因で処理が停止（デッドロック）してしまう重大な不具合を修正しました。プロセスとの通信方法を改善し、ハングアップすることなく、安定して動画の書き出しが完了するようにしました。

39. **操作性の改善 (ドラッグ操作)**:
    - **改善**: 図形の描画やリサイズ中に、マウスカーソルが動画の枠外にはみ出すと操作が強制的に終了してしまう問題を修正しました。ウィンドウ全体でドラッグを検知するように変更し、カーソルが枠外に出ても操作を継続できるようにしました。
    - **改善**: 上記の修正に加え、ドラッグ座標の制限を撤廃し、動画の枠外まで領域をはみ出して指定できるように変更しました。これにより、角丸四角形を動画の隅に配置した際に、丸みの内側で隅が隠れきらない問題を解消し、隅々まで完全に覆うことが可能になりました。

    - **操作性**: 角丸四角形の「丸み」の値を、同じレイヤー内で新しい図形を作成する際に引き継ぐように修正しました。これにより、同じ丸みの図形を複数作成する際の再設定の手間を省きました。
